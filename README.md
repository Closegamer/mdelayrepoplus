# mDelayPlusBot

## Миссия

Сервис слежения за состоянием пользователя, основанный на периодических проверках и фиксации ответов с потенциальным оповещением службы спасения в случае непредвиденных обстоятельств.

## Этапы создания бота

- FastAPI backend
- DB
- Telegram bot
- Cron
- Admin page

## Полное описание работы бота

`mDelayPlusBot` предназначен для ситуаций, когда пользователь хочет оставить заранее подготовленное сообщение и получить автоматический контрольный опрос через выбранный интервал времени.

Пользовательский сценарий:

1. Пользователь открывает Telegram-бота и отправляет команду `/start`.
2. Бот показывает главное меню:
   - `Написать новое сообщение`
   - `Прочитать свои сообщения`
3. При создании сообщения бот:
   - принимает текст;
   - предлагает период до первого опроса (включая тестовый режим);
   - отправляет данные в API.
4. API сохраняет запись в PostgreSQL:
   - данные пользователя;
   - исходный текст сообщения;
   - режим (`Реальный` или `Тестовый`);
   - индивидуальные задержки `check1/check2/check3`;
   - будущие статусы и ответы по проверкам.
5. Фоновый `cron` сервис периодически проверяет БД:
   - когда наступает время, отправляет 1-й/2-й/3-й опрос;
   - фиксирует факт отправки (`SENT`).
6. Если пользователь отвечает:
   - при ответе вида `Я в порядке` (с учетом нормализации вариантов) наблюдение завершается;
   - при любом другом ответе формируется немедленная тревога.
7. Если пользователь не ответил на все 3 опроса, после дедлайна 3-го опроса также формируется тревога.
8. Тревога отправляется в канал/чат оповещения (`ALERT_CHAT_ID`) с исходным сообщением и данными пользователя.
9. Админ-панель (Streamlit) показывает:
   - KPI (всего сообщений, пользователей, тревог, активных проверок);
   - таблицы сообщений, тревог и активных проверок;
   - ответы по этапам (`Check1`, `Check2`, `Check3`);
   - пагинацию, удаление записей, ручное обновление.

Архитектурная схема:

- `bot` — Telegram интерфейс и пользовательские сценарии.
- `api` — FastAPI слой бизнес-логики и REST API.
- `cron` — планировщик и отправка опросов/эскалаций.
- `ui` — Streamlit админ-панель мониторинга.
- `db` — PostgreSQL хранилище.
